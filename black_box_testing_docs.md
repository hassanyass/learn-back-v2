# LearnBack Black Box Testing Documentation

## 4. Learning Session Flow
The exact flow during a teaching session is as follows:

1. **Select Topic & Start Session**: Student selects a learning material and starts the teaching session.
2. **Kido asks a question**: Kido prompts the student to explain the first pedagogical point.
3. **Student answers**: Student submits their explanation via chat.
4. **Evaluator evaluates**: The invisible Evaluator LLM grades the answer (CORRECT, INCORRECT, NEEDS_INFO, IRRELEVANT) and detects misconceptions.
5. **Mastery updated**: The Bayesian Knowledge Tracing (BKT) score is mathematically updated based on the Evaluator's label. Attempt counters and Kido's memory are updated.
6. **Point Completion Check**: 
   - If the student hits the Mastery Threshold OR the Max Attempts (5) for that point, the point is marked "completed", and the system advances to the next point.
   - If all points in a topic are completed, a **Mind Map Checkpoint** is triggered, allowing the student to review/correct Kido's notes.
7. **Kido Responds**: Kido generates a character-appropriate response (or interactive widget like a Process/Comparison sorting task) to either celebrate, express confusion, or ask for more info.
8. **End session**: Once all topics are completed (or overall session BKT exceeds 95%), the session terminates.
9. **Feedback dashboard**: The student is redirected to the final feedback and analytics report.

---

## 5. BKT / Mastery Rules

**Thresholds and Initialization:**
- **Mastery threshold**: 85% (`MASTERY_THRESHOLD = 0.85`)
- **Initial mastery**: 30% (`p_init = 0.3`)
- *(Other parameters: `p_learn` = 0.2, `p_slip` = 0.1, `p_guess` = 0.2)*

**What happens after mastery?**
Once a point reaches ≥ 85% BKT score (or the student exhausts the 5 maximum attempts):
1. **Point Completed**: The current point is marked as completed in the session state.
2. **Kido Acknowledges**: Kido gracefully acknowledges understanding the point.
3. **Next Concept Unlocked**: The system advances the `current_point_index`, and Kido immediately asks an opening question about the new concept.
4. **Topic Checkpoint**: If this was the last concept in the topic, a Mind Map review phase is unlocked before proceeding to the next topic.

## 6. Feedback Dashboard Contents

The end-of-session Feedback Dashboard contains the following exact data components generated by the system:

- **Session Metadata**: Session Title, Completion Type, and Duration (in minutes).
- **Overall Mastery**: A percentage score from 0 to 100 representing the total session BKT.
- **Global Strengths**: Up to 5 high-level positive takeaways from the student's teaching.
- **Global Weak Areas**: Up to 5 areas where the student struggled.
- **Detailed Topic Breakdown**: For each topic discussed:
  - **Status & Understanding**: e.g., "completed", "partial", "skipped" / "strong" (≥70%), "good" (≥40%), "weak".
  - **Topic BKT Score**: The average mastery score for that topic.
  - **Feedback Summary**: A text summary combining the LLM-generated strengths and weaknesses for that specific topic.
  - **Topic Misconceptions**: A flat list of any incorrect assumptions made during the topic.
  - **Recommendation**: Suggested next steps or things to review.
  - **Point-by-Point Breakdown**: For every individual concept within the topic:
    - Status (completed, pending), BKT Score, Attempts made, Correctness.
    - `was_visited` and `widget_used` flags.
    - `kido_memory`: What Kido specifically learned and remembered from the student.
    - `misconceptions`: Specific errors made on that exact point.
